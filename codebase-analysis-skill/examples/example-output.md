## Sample: Algorithm Deep Dive (Excerpt)

### Voxel Map Downsampling with Running Average

**Implementation**:
[VERIFY: voxel_map.hpp:46-48]

```cpp
void down_sampling_pvec() {
    // ...
    pp.first.pnt = (pp.first.pnt * pp.second + pv.pnt) / (pp.second + 1);
    pp.first.var += pv.var;
    pp.second += 1;
}
```

**Mathematical Derivation**:

This implements the **incremental running average** formula:

$$\mu_{new} = \frac{\mu_{old} \cdot n + x_{new}}{n + 1}$$

Where:
- `pp.first.pnt` = $\mu_{new}$ (updated mean position)
- `pp.first.pnt` (before update) = $\mu_{old}$ (current mean)
- `pp.second` = $n$ (current point count)
- `pv.pnt` = $x_{new}$ (new point position)

**Why this formula?**

[VERIFY: voxel_map.hpp:42-44] - Comments indicate this avoids storing all raw points.

The running average is preferred over batch average because:
1. **Memory O(1)** — Only stores mean + count, not all points
2. **Single pass** — No need to revisit previous points
3. **Numerical stability** — For typical voxel sizes (n < 1000), this is stable
